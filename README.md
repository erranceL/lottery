# TurboFlow 线上刮刮乐

「刮开市场」——以涨/跌方向判断为主题的线上刮奖活动。用户刮开自己的目标方向，再刮开 10 组方向与奖金；方向一致赢得该格代币，刮出 TF 该格奖金翻倍，奖金兼中兼得，全部刮开后自动入账。

**在线演示**：https://errancel.github.io/lottery/

## 奖池模型（已全量验算）

| 指标 | 数值 |
| --- | --- |
| 总发行量 | 1,000,000 张 |
| 单张票面价值 | 2 代币 |
| 总派奖 | 1,000,000 代币 |
| 返奖率 | 50% |
| 中奖票 | 330,821 张（33.0821%） |
| 最高单票奖金 | 100,000 代币（5 × TF 10,000） |
| TF 事件 | 34,005 个，贡献 250,000 代币 |

奖级、中奖格数量分布与 TF 库存由精确票型矩阵（`src/engine/config.ts`）定义。每张票通过 Feistel 双射映射到唯一票型，整个奖池在生成前完全固定，可逐张审计。

```bash
npm run validate:pool   # 逐张生成 100 万张票并核对全部约束
```

## 当前版本说明（演示版）

本仓库是可静态托管的演示版，用于验收视觉、刮奖手感与奖池数学：

- `src/engine/` 是与正式版共用的奖池核心：精确票型矩阵、确定性票据生成、全量校验器。
- 发放、刮奖进度、代币入账在浏览器 localStorage 中模拟，接口形状与正式版服务端 API 对齐。
- 正式版需接入：Privy 登录、PostgreSQL 票据库、服务端派奖事务与管理后台（见产品开发计划）。

## 目录结构

```
src/engine/     奖池引擎（配置、随机、票据生成、校验）
src/store/      演示状态（发放/进度/入账，localStorage）
src/components/ 刮除区域 Canvas 组件与票面展示
src/pages/      活动页、刮奖页、历史记录
src/i18n/       中英文双语
scripts/        百万票全量校验脚本
```
